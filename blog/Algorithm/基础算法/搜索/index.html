搜索

Posted on 2021-12-04 Edited on 2022-01-26 In Algorithm , 基础算法

前言

名称 描述 最好 最坏 平均 空间 约束
线性搜索 每一个值都比较一遍 O(1) O(n) O(n) O(1) 无
二分搜索 每一次比较序列中间值 O(1) O(logn) O(1) O(1) 排序
插值搜索 每一次比较插值 O(1) O(log(logn)) O(1) O(1) 排序

列表搜索

线性搜索(Linear Search)

直接遍历

二分搜索(Binary Search)

二分查找，折半搜索

插值搜索(Interpolation Search)

利用插值公式

树、图搜索

宽度优先搜索(Breadth-First Search)

从根节点开始，沿着树的宽度遍历树的节点。从根节点开始将所有子节点加入队列。队列

深度优先搜索(Depth-First Search)

从根节点开始，沿着树的深度遍历树的节点。从根节点开始递归遍历未遍历的点。回溯

迭代深化深度优先搜索(Iterative Deepening depth-first search (IDS or IDDFS)))

重复地运行一个有深度限制的深度优先搜索，每次运行结束后，它增加深度并迭代，直到找到目标状态。

最良优先搜索(Best-First Search)

基于广度优先搜索算法，依赖于估价函数对将要遍历的节点进行估价。一个大类，包括启发式和非启发式。

双向搜索

一个从初始状态正向搜索，另一个从目标状态反向搜索，当两者在中间汇合时搜索停止

拓展算法

最短路问题

Dijkstra算法

迪杰斯特拉、戴克斯特拉，在计算机科学的人工智能等领域也被称为均一开销搜索，并被认为是最良优先搜索的一个特例。类似动态规划，从起点开始，对每一个点选择最短的到起点的路径，后面的点就能根据前面的点选择最短路径，直到终点。

堆优化版的Dijkstra算法

贝尔曼-福特算法(Bellman–Ford algorithm)

松弛操作：指对于每个顶点，都设置一个属性d[v]，用来描述从源点s到v的最短路径上权值的上界

SPFA算法(Bellman-Ford算法的优化)

Floyd-Warshall算法(Floyd-Warshall algorithm)

A * 算法(A-Star)

最良优先 + dijkstra，常用于游戏中的NPC的移动计算，或网络游戏的BOT的移动计算上，此处以自己用python写的推箱子小游戏作为测试对象，见启发式算法。

字符串搜索

KMP(Knuth-Morris-Pratt)

自己与自己先匹配，因为暴力匹配存在”匹配一段发现不匹配又要回退到起始点的问题”。

博耶-穆尔字符串搜索算法

类似KMP需要跳过字符。从后往前匹配，同时通过坏字符规则(发生不匹配字符，且此不匹配字符又在后面存在，直接将此字符对齐)和好后缀规则(发生不匹配时，最长的好后缀是否存在于不匹配字符的左侧，其他后缀组合中是否存在与P的前缀相同的情况)，尽量跳过不必要的比较。

AC自动机(Aho–Corasick)

同时与所有字典串进行匹配，通过构造一个树实现。如果每个子串互相匹配，时间复杂度会近似于匹配的二次函数。当一个字典串集合是已知的，算法的时间复杂度为输入字符串长度和匹配数量之和。

拉宾-卡普算法(Rabin–Karp algorithm)

使用散列函数以快速对每个位置能否匹配作大致的检测，此后只对通过了检测的位置进行匹配尝试。

Bitap算法（shift-or、shift-and、Baeza-Yates–Gonnet算法）

一种字符串模糊匹配算法。预处理计算一个掩码集，其中每个位代表一个字符是否在模式串中出现过，如果子串和模式串的距离小于等于给定的k，算法就认为它们是匹配的。
